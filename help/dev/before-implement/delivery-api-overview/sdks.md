# SDK

[!DNL Adobe Target] offre [SDK lato server](../../implement/server-side/server-side-overview.md) per interagire con [!DNL Target Delivery API] e facilitare l&#39;integrazione di [!UICONTROL Target] con altre soluzioni [!DNL Adobe Experience Cloud] come [!UICONTROL Adobe Analytics] e [!DNL Adobe Audience Manager].

Gli SDK ti aiutano a seguire le best practice nella gestione di sessioni e cookie. Si consiglia vivamente di utilizzare un SDK per semplificare le complessità e prevenire un utilizzo improprio dell’API.

## SDK per Node.js

Per ulteriori informazioni, consulta la [Panoramica dell’SDK per Node.js](/help/dev/implement/server-side/node-js/overview.md).

L’[SDK per Node.js](https://github.com/adobe/target-nodejs-sdk) è open source e gestito su GitHub, dove puoi trovare la documentazione e le istruzioni per l’uso.

Consulta il [registro delle modifiche](https://github.com/adobe/target-nodejs-sdk/blob/main/CHANGELOG.md) dell’SDK per Node.js.

## SDK Java

Per ulteriori informazioni, consulta la [Panoramica dell’SDK per Java](/help/dev/implement/server-side/java/overview.md).

L’[SDK per Java](https://github.com/adobe/target-java-sdk) è open source e gestito su GitHub, dove puoi trovare la documentazione e le istruzioni per l’uso.

Consulta il [registro delle modifiche](https://github.com/adobe/target-java-sdk/blob/main/CHANGELOG.md) dell’SDK per Java.

## SDK per .NET

Per ulteriori informazioni, consulta la [Panoramica dell’SDK per .NET](/help/dev/implement/server-side/net/overview.md).

L’[SDK per .NET](https://github.com/adobe/target-dotnet-sdk) è open source e gestito su GitHub, dove puoi trovare la documentazione e le istruzioni per l’uso.

Informazioni sulle [versioni](https://github.com/adobe/target-dotnet-sdk/releases) dell’SDK per .NET.
